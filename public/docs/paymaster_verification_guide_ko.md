# Grand Paymaster 시스템 검증 가이드 (Admin UI)

본 가이드는 **Grand Paymaster System**의 전체 기능(Phase 1-6)을 Admin Console을 통해 검증하는 절차를 설명합니다.

**접속 URL**: `/adminsystem/paymaster`

## 1. 대시보드 및 지표 확인 (Phase 1)
**목표**: 시스템 전체의 핵심 지표(KPI)와 상태를 확인합니다.
- [ ] **KPI 카드**: "24h Sponsor Spend(24시간 지출)", "Revenue(수익)", "Failure Rate(실패율)", "Active Chains(활성 체인 수)"에 숫자가 표시되는지 확인합니다.
- [ ] **자동 갱신**: 2~3초간 대기하며 데이터가 주기적으로 갱신되는지(숫자 변화 또는 유지) 확인합니다.

## 2. 체인 및 풀(Pool) 관리 (Phase 1 & 2)
**목표**: 개별 체인의 에이전트 상태와 가스비 잔고를 확인합니다.
- [ ] **체인 목록**: "Vision Testnet v2" (Chain ID: 1337) 등이 목록에 올바르게 표시되는지 확인합니다.
- [ ] **상태 뱃지**: 기본 상태가 초록색 `NORMAL` 인지 확인합니다.
- [ ] **잔고 확인**: "Gas Balance"가 표시되는지 확인합니다 (예: `5.0000 ETH`).
- [ ] **지출 속도**: "Spend Rate (24h)"가 표시되는지 확인합니다.

## 3. 운영 제어 기능 (Phase 1)
**목표**: 관리자 권한으로 풀을 일시정지/재개할 수 있는지 테스트합니다.
- [ ] **풀 일시정지 (Pause Pool)**:
    1. 카드 하단의 **"Pause Pool"** 버튼을 클릭합니다.
    2. 브라우저 경고창("Are you sure...?")에서 확인을 누릅니다.
    3. **검증**: 상태 뱃지가 빨간색 `PAUSED`로 바뀌고, 버튼이 "Resume Pool"로 변경되는지 확인합니다.
- [ ] **풀 재개 (Resume Pool)**:
    1. **"Resume Pool"** 버튼을 클릭합니다.
    2. **검증**: 상태 뱃지가 다시 초록색 `NORMAL`로 돌아오는지 확인합니다.

## 4. 라이브 시스템 통합 및 자율 회복 (Phase 6 Demo)
**목표**: `PaymasterAgent`의 자율 로직과 프론트엔드-백엔드 연동을 검증합니다.

**시나리오**: 치명적인 "잔고 부족(Low Balance)" 상황을 시뮬레이션하여 에이전트가 `SAFE_MODE`로 전환되는지 확인합니다.

1.  **초기 상태**: 체인 상태가 `NORMAL` 인지 확인합니다.
2.  **Drain 트리거 (장애 유발)**:
    *   카드 하단의 보라색 **"Sim: Drain"** 버튼을 클릭합니다.
    *   *동작*: 이 버튼은 모의 DB의 잔고를 `0.1 ETH` (최소 유지 잔고 `1.0 ETH` 미만)로 강제 변경합니다.
3.  **관찰 및 검증**:
    *   약 2~3초(에이전트 주기) 대기합니다.
    *   **검증**: 상태 뱃지가 자동으로 노란색 **`SAFE_MODE`**로 변경되는지 확인합니다.
    *   **검증**: "Gas Balance" 텍스트가 **빨간색**으로 변하고 `0.1000 ETH`로 표시되는지 확인합니다.

## 5. 보안 및 로그 (Phase 5)
**목표**: 운영 로그 확인.
- [ ] **Logs 버튼**: 체인 카드의 "Logs" 버튼을 클릭합니다.
    - *(현재는 개발자 콘솔(F12)에 "[PaymasterAgent] Mode Transition" 로그가 출력됩니다.)*

---
**검증 성공 기준**:
위의 **4번 시나리오 (Drain -> Safe Mode -> UI 업데이트)**가 페이지 새로고침 없이 실시간으로 동작한다면, **시스템 통합(Phase 6)** 및 **자율 에이전트 로직(Phase 2)**이 성공적으로 검증된 것입니다.
