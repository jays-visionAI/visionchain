# Vision Chain 노드 분류 체계 (Node Taxonomy) v1.0

Vision Chain 네트워크의 안정성과 유동적인 유저 참여를 동시에 수용하기 위한 4단계 노드 분류 체계입니다.

---

## 1. 노드 타입별 비교 요약

| 노드 종류 | 가동성 요구 (Uptime) | 운영 주체 | 핵심 역할 | 주요 인센티브 |
|:---:|:---:|:---:|:---|:---|
| **Authority Node** | **99.9% (24/7)** | 재단/파트너 | 네트워크 최종성(Finality) 보장 | 운영 실비 및 에코시스템 펀드 |
| **Consensus Node** | **95% 이상** | 커뮤니티(PoS) | 블록 생성 및 네트워크 분산화 | VCN 블록 보장 및 스테이킹 보상 |
| **Agent Node** | **가변적 (온디맨드)** | 서비스 운영사 | Paymaster 대납 및 데이터 인덱싱 | 서비스 수수료 (Surcharge) |
| **Edge/Light Node** | **간헐적 (자유)** | 일반 유저 | 로컬 검증 및 프라이버시 유지 | 참여 기여 보상 (Micro-rewards) |

---

## 2. 상세 역할 정의

### 2.1 Authority Node (재단 노드)
- **정의**: 네트워크 런칭 초기부터 시스템의 생존을 책임지는 노드입니다.
- **가동성**: 클라우드 서버에서 24/7 상시 대기해야 합니다.
- **제약**: 오프라인 발생 시 네트워크 전체 가동성에 치명적이므로 슬래싱보다 더 엄격한 관제 대상입니다.

### 2.2 Consensus Node (검증 노드)
- **정의**: VCN을 스테이킹하고 블록 생성에 참여하는 표준 PoS 노드입니다.
- **가동성**: 높은 가동률(High Availability)이 권장되나, 간헐적 다운타임은 슬래싱(보상 삭감)을 통해 대응합니다.
- **전략**: 사용자가 노드를 꺼두면 그 시간만큼의 이자를 받지 못하며, 너무 오래 꺼두면 밸리데이터 리스트에서 제거됩니다.

### 2.3 Agent Node (수수료 대납 노드)
- **정의**: Grand Paymaster 등 특정 서비스 로직을 수행하는 특수 목적 노드입니다.
- **가동성**: **가장 유동적**입니다. 자신이 담당하는 dApp의 트랜잭션이 있을 때만 켜져 있어 보상을 받거나, 상시 대기하여 쿼리를 처리할 수 있습니다.
- **수익 모델**: 사용자 트랜잭션을 대신 실행해 준 대가로 부가 수수료를 받습니다.

### 2.4 Edge Node (참여 노드)
- **정의**: 일반 유저의 PC나 스마트 기기에서 가볍게 돌아가는 노드입니다.
- **가동성**: 제한이 없습니다. 유저가 게임을 하거나 앱을 켤 때만 노드가 활성화되어 데이터를 검증하고 네트워크 대역폭에 기여합니다.
- **보상**: 미미하지만 참여 시간이나 기여도에 비례한 VCN을 '에어드랍'이나 '팀' 형태로 지급받습니다.

## 3. 간헐적 운영 효율화 전략 (Intermittent Support)

질문하신 "유저가 노드를 껐다 켤 때"를 위해 다음과 같은 메커니즘을 설계합니다.

1.  **Checkpoint Sync**: 노드를 새로 켰을 때 전체 블록을 다 다운받지 않고, 가장 최근의 '체크포인트'부터 즉시 동기화하여 가동 시간을 단축합니다.
2.  **Snapshot Storage**: 꺼져 있는 동안의 데이터를 다른 노드들로부터 P2P로 빠르게 요약본만 전달받는 기술을 적용합니다.
3.  **Active List Switching**: 노드가 오프라인이 되면 즉시 '활성 리스트'에서 제외하여 합의 정족수 계산에서 무시하고, 다시 들어오면 리스트에 복귀시키는 동적 합의(Dynamic Consensus)를 지향합니다.
